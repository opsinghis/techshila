## The Problem with AI-Assisted Development Today

Most teams using AI for coding are stuck in **ad-hoc prompting** — inconsistent results that depend on who writes the prompt. There's no traceability from Jira ticket to pull request, tests get skipped, and decisions live in the developer's head instead of the repo.

**The result:** AI tools boost individual speed but create organisational risk — no consistency, no traceability, no safety net.

## Our Framework: Three Pipelines, One System

Agentic SDLC is built on two core architectural principles:

**LISA (Layered Information State Architecture)** — each phase reads the previous state, writes its own. Markdown artifacts become the single source of truth with a full audit trail for every decision.

**Ralph Loops (Idempotent Retry with Signal-Based Routing)** — every phase emits a signal: ADVANCE, BLOCK, LOOPBACK, or TERMINAL. Max 3 retries per loop — no infinite spirals.

### 1. Story Readiness Pipeline

Three sequential commands that turn a Jira ticket into a fully-engineered, test-ready story:

- **/story-draft** — generates acceptance criteria, edge cases, and initial scope
- **/story-engineer** — adds technical design, file impact analysis, and implementation notes
- **/story-qa** — creates test scenarios, defines coverage requirements, and validates readiness

### 2. Development Workflows

Purpose-built pipelines for every type of work:

- **Bug-Fix** (16 phases) — reproduce, root-cause, fix, test, PR
- **Feature-Enhance** (17 phases) — extend existing functionality with backward compatibility checks
- **Feature-Build** (20+ phases) — greenfield features with full architecture design
- **SDET Router** — automatically classifies your task and selects the right workflow

### 3. QE Pipeline

Six automated testing categories in every run:

- **Functional** — unit + component tests
- **Integration** — service boundary tests
- **E2E** — full user journey tests
- **Performance** — load + response time benchmarks
- **Security** — OWASP + dependency audit
- **Accessibility** — WCAG compliance checks

Plus: failure classification (env vs flaky vs real), auto-created Jira bugs with repro steps, and re-run intelligence that only retests affected areas.

## Getting Started

The framework is open-source. Three steps to get running:

1. Copy the `.claude/` directory into your repo
2. Run the generators: `/generate-development-workflows`, `/generate-story-workflows`, `/generate-qe-workflows`
3. Execute your first workflow: `/bug-fix PROJ-123`

**Recommended adoption path:** Week 1 — Story Readiness on your next ticket. Week 2 — Bug-Fix on a real production bug. Week 3 — Feature builds + QE pipelines.
